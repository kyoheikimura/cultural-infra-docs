    Linux Command Reference（Linux コマンドリファレンス）¶

    Linux サーバ運用でよく使うコマンドを、カテゴリ別に表形式でまとめる。

    🧩 基本操作¶

    コマンド 説明
    pwd 現在のディレクトリを表示
    ls -l 詳細表示で一覧
    ls -la 隠しファイル含め一覧
    cd /path ディレクトリ移動
    cd .. 1 つ上へ
    cd ~ ホームへ

    🧩 ファイル操作¶

    コマンド 説明
    cat file ファイル内容表示
    less file ページング表示
    tail -n 50 file 末尾 50 行表示
    cp file1 file2 コピー
    cp -r dir1 dir2 ディレクトリコピー
    mv old new 移動 / 名前変更
    rm file 削除
    rm -r dir ディレクトリ削除
    touch file 空ファイル作成
    mkdir dir ディレクトリ作成
    mkdir -p a/b 階層ごと作成

    🧩 権限・所有者¶

    コマンド 説明
    chmod 644 file 読み書き権限設定
    chmod 700 dir 所有者のみアクセス
    chown user:group file 所有者変更
    chown -R user:group dir 再帰的に変更

    🧩 プロセス管理¶

    コマンド 説明
    ps aux 全プロセス表示
    ps aux \| grep nginx プロセス検索
    kill <PID> プロセス終了
    kill -9 <PID> 強制終了

    🧩 systemd（サービス管理）¶

    コマンド 説明
    systemctl start nginx サービス起動
    systemctl stop nginx 停止
    systemctl restart nginx 再起動
    systemctl status nginx 状態確認
    systemctl enable nginx 自動起動 ON
    systemctl disable nginx 自動起動 OFF

    🧩 ネットワーク¶

    コマンド 説明
    ip a IP アドレス確認
    ip route ルーティング確認
    ping 8.8.8.8 接続確認
    ping google.com DNS + 接続確認
    dig example.com DNS 詳細
    nslookup example.com DNS 簡易確認
    ss -tulnp ポート一覧

    🧩 ディスク・メモリ¶

    コマンド 説明
    df -h ディスク使用量
    du -sh /var/log ディレクトリ容量
    free -h メモリ使用量

    🧩 ログ調査¶

    コマンド 説明
    journalctl -u ssh SSH のログ
    journalctl -u nginx nginx のログ
    journalctl -xe エラー詳細
    tail -f /var/log/syslog syslog 監視
    tail -f /var/log/auth.log 認証ログ監視

    🧩 SSH / SCP¶

    コマンド 説明
    ssh user@host SSH 接続
    ssh -i key user@host 鍵指定で接続
    scp file user@host:/path ファイル転送
    scp -r dir user@host:/path ディレクトリ転送

    🧩 パッケージ管理（Ubuntu）¶

    コマンド 説明
    apt update パッケージ情報更新
    apt upgrade -y アップデート
    apt install nginx インストール
    apt remove nginx 削除

    🧩 ユーザ管理¶

    コマンド 説明
    adduser mike ユーザ追加
    usermod -aG sudo mike sudo 権限付与
    passwd mike パスワード変更

    🧩 ファイル検索¶

    コマンド 説明
    find / -name "file.txt" ファイル検索
    grep -R "keyword" /path 文字列検索

    🧩 まとめ¶

    Linux の基本操作は、Ansible や GitOps の基盤となる。
    このテーブル版リファレンスは、最速でコマンドを思い出せるように最適化している。
