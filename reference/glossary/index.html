    Glossary（用語集）¶

    IaC / Ansible / GitOps / Linux 運用で頻繁に登場する用語をまとめる。
    各項目は実務での意味・使い方を簡潔に説明。

    🧩 基本概念¶

    用語 説明
    IaC（Infrastructure as Code） インフラ構成をコードで管理する考え方。再現性・自動化・レビューが可能になる。
    GitOps Git リポジトリを単一の真実のソースとして、インフラを自動的に同期・デプロイする運用モデル。
    Playbook Ansible の実行定義ファイル。タスクの順序・条件・対象を記述する。
    Role Playbook を構造化する単位。タスク・テンプレート・変数を分離して再利用性を高める。
    Task Ansible の最小実行単位。モジュールを呼び出して設定や操作を行う。
    Handler 特定のイベント（例：設定変更）後に実行されるタスク。notify で呼び出す。

    🧩 SSH / セキュリティ関連¶

    用語 説明
    SSH（Secure Shell） 暗号化された通信でリモート接続を行うプロトコル。鍵認証が推奨。
    公開鍵認証 秘密鍵と公開鍵のペアで認証する方式。パスワードより安全。
    sudo 一時的に管理者権限を付与してコマンドを実行する。NOPASSWD 設定で自動化可能。
    root Linux の管理者ユーザ。直接ログインは避け、sudo 経由で操作するのがベストプラクティス。

    🧩 Git / バージョン管理¶

    用語 説明
    commit 変更を履歴として保存する操作。メッセージで内容を明示する。
    branch 開発ラインを分岐させる仕組み。main は常に安定状態を維持。
    merge ブランチの変更を統合する操作。レビュー後に行う。
    pull / push リモートとの同期操作。pull は取得、push は送信。
    tag 特定のコミットにラベルを付ける。リリース管理に使用。

    🧩 Semaphore / GitOps 実行基盤¶

    用語 説明
    Semaphore Ansible の GUI 実行環境。ジョブ管理・Runner・Git 連携を提供。
    Runner Semaphore が Ansible を実行するためのエージェント。SSH 鍵を用いて接続。
    Inventory 対象ホストの一覧。グループ化して Playbook に渡す。
    Environment Semaphore の変数セット。Playbook 実行時に注入される。
    Template ジョブ実行設定。Playbook・Inventory・Environment を紐付ける。

    🧩 ネットワーク / インフラ¶

    用語 説明
    CIDR IP アドレス範囲を表す表記法。例：192.168.1.0/24。
    Gateway ネットワークの出口。外部通信の経路。
    DNS ドメイン名を IP に変換する仕組み。
    netplan Ubuntu のネットワーク設定ツール。YAML 形式で構成を定義。
    VLAN 仮想ネットワーク。物理的に分離せず論理的に区分けする。

    🧩 ドキュメント / 運用¶

    用語 説明
    README.md プロジェクト概要・構成・使い方を記載するファイル。
    Reference コマンドや設定のリファレンス集。運用時の辞書として利用。
    Glossary 用語集。チーム内で共通理解を持つための基盤。
    Docs フォルダ構成 ansible/, semaphore/, reference/ に分けて整理するのが推奨。

    🧩 まとめ¶

    この用語集は、IaC 運用における共通言語を整理したもの。
    チーム内での理解統一・ドキュメント作成・教育資料として活用できる。
