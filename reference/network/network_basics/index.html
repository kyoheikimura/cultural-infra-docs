    Network Basics（ネットワーク基礎リファレンス）¶

    サーバ運用・Ansible・仮想化環境（Proxmox / Hyper-V）で頻繁に使う
    ネットワークの基本コマンド・概念・調査方法をまとめる。

    🧩 IP / インターフェース確認¶

    コマンド 説明
    ip a IP アドレス・NIC 状態を確認
    ip link インターフェース一覧
    ip -br a 簡易表示
    ip addr show eth0 特定 NIC の詳細

    🧩 ルーティング¶

    コマンド 説明
    ip route ルーティングテーブル
    ip route get 8.8.8.8 どの経路で出るか確認
    ip r 短縮表示

    🧩 DNS¶

    コマンド 説明
    dig example.com DNS 詳細
    dig +short example.com IP のみ
    nslookup example.com 簡易 DNS
    host example.com 名前解決

    🧩 接続確認（Ping / Traceroute）¶

    コマンド 説明
    ping 8.8.8.8 ICMP 到達確認
    ping google.com DNS + ICMP
    traceroute 8.8.8.8 経路確認
    mtr 8.8.8.8 traceroute + ping（連続）

    🧩 ポート確認（ss / netstat）¶

    コマンド 説明
    ss -tulnp LISTEN 中のポート一覧
    ss -tnp TCP のみ
    ss -uanp UDP のみ
    netstat -tulnp 旧コマンド（互換）

    🧩 ファイアウォール（UFW）¶

    コマンド 説明
    ufw status 状態確認
    ufw allow 22/tcp SSH 許可
    ufw allow 80/tcp HTTP 許可
    ufw enable 有効化

    🧩 CIDR（サブネット）¶

    CIDR ホスト数 用途
    /24 254 小規模 LAN（一般的）
    /25 126 分割された LAN
    /26 62 VLAN 用
    /16 65534 大規模ネットワーク

    ✔ よく使う計算例¶

    表記 意味
    192.168.1.0/24 192.168.1.x 全体
    192.168.1.10/32 単一ホスト
    10.0.0.0/16 10.0.x.x

    🧩 ARP / MAC アドレス¶

    コマンド 説明
    ip neigh ARP テーブル
    arp -a 旧コマンド
    ip link show MAC アドレス確認

    🧩 ネットワークサービスの調査¶

    コマンド 説明
    curl -I http://example.com HTTP ヘッダ確認
    curl -v 詳細ログ
    nc -zv host 22 ポート疎通確認
    telnet host 25 SMTP などの確認

    🧩 netplan（Ubuntu）¶

    ✔ 設定ファイル¶

    項目 内容
    設定ファイルパス /etc/netplan/*.yaml
    説明 Netplan の設定ファイルが格納されるディレクトリ
    形式 YAML 形式でネットワーク構成を定義

    ✔ 適用¶

    コマンド 説明
    netplan try 変更テスト（90 秒で戻る）
    netplan apply 永続反映
    netplan generate 設定生成

    🧩 VLAN（仮想化環境でよく使う）¶

    コマンド 説明
    ip link add link eth0 name eth0.10 type vlan id 10 VLAN10 作成
    ip a VLAN インターフェース確認
    ip link delete eth0.10 VLAN 削除

    🧩 Proxmox / Hyper-V でよくある調査¶

    コマンド 説明
    ip a VM の NIC 状態確認
    ip route ゲートウェイ確認
    ping <gateway> VLAN / ブリッジ疎通確認
    ss -tulnp VM 内サービス確認

    🧩 トラブルシュート（よくある原因）¶

    症状 原因 対処
    ping は通るが名前解決できない DNS 設定ミス /etc/resolv.conf を確認
    SSH できない ポート閉塞 / 鍵不一致 ss -tulnp / authorized_keys
    外へ出られない デフォルトゲートウェイ不正 ip route
    VLAN が通らない PVE / Hyper-V 側のタグ設定ミス ブリッジ設定を確認
    netplan apply で落ちる YAML インデントミス netplan try で検証

    🧩 まとめ¶

    ネットワークはサーバ運用・Ansible・仮想化の基盤であり、
    このリファレンスは 最速で調査・確認ができるように最適化している。
