    Inventory Design（インベントリ設計）¶

    このドキュメントでは、Ansible のインベントリ構成と変数管理の設計方針をまとめる。
    本環境では /opt/ansible/inventory/prod.yml をメインインベントリとして運用し、
    group_vars と host_vars による変数分離を採用している。

    🎯 設計目的¶

    • サーバ群を論理的に分類し、Playbook 実行範囲を明確化する
    • 環境（prod / stg / qa）ごとに設定を分離する
    • 冪等性を維持しつつ、変数の重複を排除する
    • GitHub 上で差分管理しやすくする

    📁 ディレクトリ構成¶

    /opt/ansible
├── inventory/
│   ├── prod.yml
│   ├── stg.yml
│   └── qa.yml
├── group_vars/
│   ├── all.yml
│   ├── linux.yml
│   └── windows.yml
└── host_vars/
├── lab-ai-01.yml
├── lab-db-01.yml
└── lab-vision-01.yml

    🧩 インベントリファイル（例）¶

    inventory/prod.yml 

    all:
  children:
    lab:
      hosts:
        lab-ai-01:
          ansible_host: 192.168.1.151
        lab-db-01:
          ansible_host: 192.168.1.161
        lab-vision-01:
          ansible_host: 192.168.1.171
        lab-batch-01:
          ansible_host: 192.168.1.181
        lab-ui-01:
          ansible_host: 192.168.1.191
        lab-llm-01:
          ansible_host: 192.168.1.194

    グループ変数（group_vars）¶

    group_vars/linux.yml 

    ansible_user: mike
ansible_become: true
ansible_become_method: sudo
ansible_become_user: root
ansible_python_interpreter: /usr/bin/python3
    目的： - Linux 共通設定を一括管理 - SSH / sudo / Python の設定を統一

    ホスト変数（host_vars）¶

    host_vars/lab-ai-01.yml 

    netplan_interface_name: ens18
ip_address: 192.168.1.151
gateway: 192.168.1.1
dns_servers:
  - 192.168.1.1
    目的： - 各ノード固有のネットワーク設定を保持 - netplan ロールで自動反映

    環境分離の考え方¶

    環境 ファイル 主な用途
    prod inventory/prod.yml 本番環境
    stg inventory/stg.yml 検証環境
    qa inventory/qa.yml テスト環境

    Playbook 実行時に -i inventory/prod.yml を指定することで、 環境ごとの設定を切り替える。

    🧩 ベストプラクティス¶

    • 環境ごとに inventory ファイルを分ける
    • 共通設定は group_vars/all.yml に集約する
    • ホスト固有設定は host_vars に分離する
    • 変数名はロール内のテンプレートと一致させる
    • IP・DNS・gateway は変数化して冪等性を確保する

    🧩 運用ルール¶

    • 新しいサーバを追加する場合は、まず inventory/prod.yml に登録
    • 次に host_vars/.yml を作成
    • 最後に ansible-playbook -i inventory/prod.yml playbooks/site.yml --check で検証
    • 問題なければ commit → push → Semaphore で自動反映

    🧩 トラブルシュート¶

    症状 原因 対処
    unreachable=1 IP アドレス誤り host_vars の修正
    failed=1 sudo 権限なし group_vars/linux.yml の ansible_become 設定確認
    changed=0 ロール未適用 Playbook の tags 設定確認
