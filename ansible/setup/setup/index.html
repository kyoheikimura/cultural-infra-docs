    Ansible サーバ構築手順（lab-ansible-01）¶

    このドキュメントでは、Ansible 管理サーバ（lab-ansible-01）の構築手順をまとめる。
    本環境は /opt/ansible を Single Source of Truth とし、GitHub と同期して IaC を実現する。

    1. Ubuntu 初期設定¶

    パッケージ更新¶

    apt update && apt upgrade -y

    必要パッケージ¶

    apt install -y git curl vim python3 python3-pip

    2. Ansible インストール¶

    ansible-core のインストール¶

    pip install ansible-core

    バージョン確認¶

    ansible --version

    3. ディレクトリ構成の準備¶

    mkdir -p /opt/ansible
cd /opt/ansible

    4. GitHub リポジトリとの連携¶

    SSH キー生成¶

    ssh-keygen -t ed25519 -C "ansible-server"

    公開鍵の確認¶

    cat ~/.ssh/id_ed25519.pub
    → GitHub の Deploy Key に登録（Read/Write）

    リポジトリの clone¶

    git clone git@github.com:<your-account>/<repo>.git /opt/ansible

    5. ansible.cfg の設定¶

    /opt/ansible/ansible.cfg 

    [defaults]
inventory = inventory
host_key_checking = False
retry_files_enabled = False
roles_path = playbooks/roles
interpreter_python = auto_silent

    6. SSH 公開鍵の配布（管理対象ノード）¶

    ansible サーバから各ノードへ¶

    ssh-copy-id mike@192.168.1.xxx
※ 全ノードに実施  
※ パスワード入力は初回のみ

    7. Inventory の作成¶

    inventory/prod.yml

    all:
  children:
    lab:
      hosts:
        lab-ai-01:
          ansible_host: 192.168.1.151
        lab-db-01:
          ansible_host: 192.168.1.161
        lab-vision-01:
          ansible_host: 192.168.1.171
        lab-batch-01:
          ansible_host: 192.168.1.181
        lab-ui-01:
          ansible_host: 192.168.1.191
        lab-llm-01:
          ansible_host: 192.168.1.194

    8. 接続確認¶

    ansible all -m ping -i inventory/prod.yml

    9. Playbook 実行テスト¶

    ansible-playbook -i inventory/prod.yml playbooks/site.yml --check

    10. GitHub との同期（運用）¶

    変更を取り込む¶

    git pull origin main

    変更を反映する¶

    ansible-playbook -i inventory/prod.yml playbooks/site.yml

    11. GitOps（Semaphore）との連携（概要）¶

    GitHub → Semaphore → 自動デプロイ

    • Deploy Key を Semaphore に登録
    • Playbook を GUI から実行可能
    • ログが自動保存される 詳細は gitops_semaphore.md を参照。

    12. トラブルシュート¶

    • known_hosts の削除
    • 公開鍵の再配布
    • sshd_config の確認
