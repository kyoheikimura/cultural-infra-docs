    Semaphore Key Store Design（鍵管理ガイド）¶

    このドキュメントでは、Semaphore における SSH 鍵・Deploy Key・環境変数の管理方法をまとめる。
    GitHub（SSOT）・Semaphore（GitOps 実行基盤）・Ansible（構成管理）の 3 層構造を前提とする。

    🎯 Key Store の目的¶

    • Ansible 実行に必要な SSH 秘密鍵を安全に保管する
    • GitHub リポジトリを pull するための Deploy Key を管理する
    • Playbook 実行時に利用する 環境変数を一元管理する
    • Runner が鍵を読み取れるように 適切な権限で保存する

    📁 Key Store の構成¶

    Semaphore の Key Store は以下の 3 種類の鍵を扱う：

    種類 用途 保存場所
    SSH Private Key 管理対象サーバへの接続 /etc/semaphore/keys/
    SSH Public Key 管理対象サーバの authorized_keys GitHub / Ansible ロール
    Deploy Key GitHub リポジトリの pull GitHub → Deploy Keys 
    /etc/semaphore/keys/mike_key
/etc/semaphore/keys/mike_key.pub

    ✔ 権限（最重要）¶

    ホスト側のディレクトリ：/opt/semaphore/keys

    は以下の権限である必要がある： 

    chown 1001:0 /opt/semaphore/keys
chmod 700 /opt/semaphore/keys

    Semaphore のコンテナユーザは： 

    uid=1001
gid=0

    なので、これが一致していないと鍵が保存されない。

    🧩 2. Deploy Key（GitHub リポジトリ用）¶

    ✔ GitHub 側の設定¶

    GitHub → Repository → Settings → Deploy Keys

    • Title: semaphore-deploy-key
    • Key: id_rsa.pub（Semaphore の公開鍵）
    • Allow write access: ON（必須）

    ✔ Semaphore 側の設定¶

    Repository → New Repository Repository Type: Git URL: git@github.com:/.git Branch: main SSH Key: semaphore-deploy-key

    これにより、Semaphore が GitHub から最新の Playbook を pull できる。

    🧩 3. Environment Variables（環境変数）¶

    Environment → New Environment

    例： 

    ANSIBLE_HOST_KEY_CHECKING=False
ANSIBLE_STDOUT_CALLBACK=yaml

    SSH 鍵のパスは以下のように設定する： 

    ansible_ssh_private_key_file=/etc/semaphore/keys/mike_key

    🧩 4. Key Store の運用ルール¶

    ✔ 原則¶

    • 鍵は Semaphore の Key Store にのみ保存
    • Playbook やリポジトリに鍵を置かない
    • 鍵の更新は PR ではなく Key Store で実施
    • Deploy Key は GitHub 側で管理
    • SSH 公開鍵は Ansible ロール（users ロール）で配布

    ✔ 禁止事項¶

    • Playbook 内に鍵を埋め込む
    • GitHub に秘密鍵を push
    • Semaphore コンテナ内で手動編集
    • root で鍵を上書き

    🧩 5. トラブルシュート¶

    症状 原因 対処
    Key Store に鍵が保存されない /opt/semaphore/keys の権限不一致 chown 1001:0 を実行
    SSH 接続に失敗 秘密鍵のパスが不正 Environment の設定を確認
    Git pull に失敗 Deploy Key の権限不足 GitHub の “Allow write access” を確認
    公開鍵が反映されない users ロール未実行 --tags users で反映

    🧠 ベストプラクティス¶

    • SSH 鍵は ed25519 を推奨
    • Deploy Key は リポジトリごとに分ける
    • Key Store のバックアップは /opt/semaphore/keys を丸ごと保存
    • 鍵のローテーションは 半年〜1年に1回
    • users ロールで公開鍵を配布し、秘密鍵は Key Store のみに置く
