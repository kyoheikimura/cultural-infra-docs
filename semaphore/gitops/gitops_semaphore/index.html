    Ansible Operations Guide（運用ガイド）¶

    このドキュメントでは、Ansible を日常運用する際のルール・手順・ベストプラクティスをまとめる。
    本ガイドは GitHub（SSOT）・Semaphore（GitOps 実行基盤）・Ansible（構成管理）の 3 層構造を前提とする。

    🎯 運用の基本方針¶

    • GitHub を唯一の真実のソース（SSOT）とする
    • Playbook / ロールの変更は PR → review → merge
    • Semaphore は main ブランチを pull して実行
    • ローカルでの直接実行は原則禁止（緊急時のみ）
    • 冪等性を最優先し、毎回安全に実行できる状態を維持する

    📁 運用対象のディレクトリ構造¶

    /opt/ansible/
├── ansible.cfg
├── inventory/
├── playbooks/
├── roles/
└── docs/

    🧩 1. Playbook 実行ルール¶

    ✔ 基本実行（全ロール）¶

    ansible-playbook playbooks/site.yml

    ✔ tags を使った部分実行¶

    • SSH 設定だけ反映

      ansible-playbook playbooks/site.yml --tags ssh

    • ネットワークだけ反映

      ansible-playbook playbooks/site.yml --tags netplan

    • ユーザ管理だけ反映

      ansible-playbook playbooks/site.yml --tags users

    ✔ 冪等性チェック（dry-run）¶

    ansible-playbook playbooks/site.yml --check

    🧩 2. 変数管理（group_vars / host_vars）¶

    ✔ 原則¶

    • 変数は Playbook に書かない
    • ロール内に書かない
    • すべて group_vars/ または host_vars/ に集約する

    ✔ 例：lab グループ¶

    group_vars/lab.yml

    ✔ 例：個別ホスト¶

    host_vars/lab-ai-01.yml

    🧩 3. SSH / sudo / ユーザ管理の運用¶

    ✔ SSH 公開鍵認証のみ¶

    • パスワードログイン禁止
    • root SSH ログイン禁止
    • AllowTcpForwarding / PermitTunnel はロールで統一管理

    ✔ sudo 運用¶

    • mike ユーザに NOPASSWD
    • sudoers はロールでテンプレート化
    • 手動編集は禁止

    🧩 4. ロール追加時のルール¶

    ✔ 新規ロール作成手順¶

    1. roles/<role_name>/ を作成
    2. tasks/main.yml を作成
    3. templates/ or files/ を配置
    4. handlers/main.yml を作成
    5. docs/ansible/roles/<role_name>/<role_name>_role.md を作成
    6. site.yml にロールを追加
    7. PR → review → merge

    ✔ ロールの責務は単一にする¶

    • SSH → SSH のみ
    • netplan → ネットワークのみ
    • users → ユーザ管理のみ
    • common → OS 基本設定のみ

    🧩 5. GitOps（Semaphore）運用ルール¶

    ✔ 実行フロー¶

    1. GitHub に push
    2. Semaphore が自動で pull
    3. テンプレートを実行
    4. ログを確認
    5. 成功したら完了

    ✔ Semaphore での実行禁止事項¶

    • Playbook の直接編集
    • Inventory の直接編集
    • SSH 鍵の手動追加
    • main 以外のブランチ実行

    ✔ テンプレート例¶

    • site.yml 実行テンプレート
    • --tags ssh 用テンプレート
    • --tags netplan 用テンプレート

    🧩 6. トラブルシュート¶

    症状 原因 対処
    netplan が毎回 changed YAML の空白差分 テンプレート整形
    SSH が restart 連発 テンプレ差分 sshd_config.j2 を確認
    Semaphore が pull 失敗 Deploy Key 権限不足 GitHub の設定確認
    Playbook が host を見つけない inventory の書き間違い inventory/ を確認

    🧠 7. ベストプラクティス¶

    • Playbook は ロジックを持たず、ロール呼び出しだけ
    • handler は テンプレート変更時のみ 実行
    • when: 条件は最小限に
    • shell: / command: は極力使わない
    • すべての設定は テンプレート化
    • すべての変更は GitHub → Semaphore の流れで反映
