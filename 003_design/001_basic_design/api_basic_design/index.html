    API基本設計（API Basic Design）¶

    1. 目的¶

    UI・AI・Data を接続する API の基本仕様を定義する。

    2. API一覧¶

    API 概要
    POST /mood 気分データ登録
    GET /mood/{id} mood取得
    POST /analysis LLM解析
    POST /vision Vision解析
    POST /embed 埋め込み生成
    GET /result/{id} 結果取得

    3. 認証方式¶

    • JWT（将来拡張可能）
    • APIキー（内部通信）

    4. ステータスコード方針¶

    • 200：成功
    • 400：入力エラー
    • 500：AI/DBエラー

    5. 共通レスポンス形式¶

    {
  "status": "success",
  "data": {},
  "meta": {}
}
